What is MCP?

MCP is an open protocol that defines how AI systems can interact with external tools and data sources. Think of it as the USB of the AI world – just as USB standardizes connections between devices, MCP standardizes how AI connects to business systems.

The protocol defines:

  • Standard interfaces for tool discovery and invocation
  • Security model with authentication and authorization

Building MCP Servers: Practical Guide

Step 1: Define Your Tools

Based on the bootcamp-resources agent engineering curriculum, start with clear tool definitions:

// Define available tools in MCP server config
{
  "tools": [
    {
      "name": "query_database",
      "description":"Query PostgreSQL database for customer data",
      "inputSchema": {
        "type": "object",
        "properties": {
          "query": {"type":"string"},
          "limit": {"type":"integer"}
        }
    }
  ]
}

Step 2: Implement Tool Handlers

async function query_database(input, env) {
  // Connect to database using env credentials
  const client = new PostgreSQL(env.DB_CONNECTION);
  await client.connect();

  // Execute query safely (prevent injection)
  const result = await client.query(
    `SELECT * FROM ${table} WHERE ${condition} LIMIT $1`,
    [input.limit]
  );

  // Return formatted results
  return {
    "data": result.rows.map(const row => ({
      "id": row.id,
      "values": row
    }))
  };
}

Step 3: Expose Tools via MCP Protocol

// Server initialization (simplified from rag-python patterns)
const mcpServer = new MCPServer({
  "name": "business-database-server",
  "version": "1.0.0",
  "tools": [query_database, get_document, search_email]
});

// Start server on port 3000
await mcpServer.listen(3000);

MCP in Action: Real-World Examples

From our analysis of typescript-next-starter, here are actual MCP use cases:

Example 1: AI + Database Integration

Before MCP: To query a database, you'd need a custom API, authentication, data serialization, and error handling. For each database, you'd write custom code.

With MCP: Define a generic "query_tool" and connect it to your databases. The AI can now query across PostgreSQL, MySQL, MongoDB, and Redis using the same interface.

Example 2: AI + Business Logic

Before MCP: To create an AI assistant that can actually update Salesforce, you'd need to build a custom Salesforce integration with OAuth, data mapping, and API limits.

With MCP: Add a Salesforce MCP server, and suddenly your AI can query and potentially update your CRM data securely with proper access controls.
